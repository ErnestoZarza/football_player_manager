AIMS:

  • To analyse our converter tasks and backend datasets in order to create insights that show us some relevant properties inside the data.

  • Use the above results to mesure performance and help the converter decision making in the queueing process.


Environments:

  • We analysed the results for the environments "dev", "staging", "production".


Analyses:

  • by Source extension (Analysis based on the file type of the converted files).

  • by Single File (Analysis based on a single file type. You can remove outliers and other actions).

  • by Size (Analysis based on the size of the converted files).

  • by Company (Analysis based on data from our clients' companies).

  • by User (Analysis based on our customers' user data).

  • by Session (Analysis based on our clients' session data).

  • by Months (Analysis based on data from the last 6 months).

  • by Weekdays (Analysis based on data grouped by days of the week).

  • by Hours (Analysis based on data grouped by time of day).



URL's:

  • "Get "http://0.0.0.0:8000/analyses/*entity*/*analysis*/*environment*/' + ?name="name"
  • Get "http://0.0.0.0:8000/analyses/*entity*/*environment*/' + ?name="name"

variables:

  • entity: 'session', 'user', 'company', 'weekly', 'daily', 'source_extension', 'monthly', 'size'
  • analysis: 'session', 'user', 'company', 'weekly', 'daily', 'source_extension', 'monthly', 'size'
  • environment: 'dev', 'staging', 'production'


METHODS:

  • Combining the Python Pandas libraries for data analysis and Matplotlib to visualise the results and create graphics and tables.
